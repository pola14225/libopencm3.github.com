16 Each I/O port has 16 individually configurable bits. Many I/O pins share GPIO
17 functionality with a number of alternate functions and must be configured to
18 the alternate function mode if these are to be accessed. A feature is available
19 to remap alternative functions to a limited set of alternative pins in the
20 event of a clash of requirements.
21 
22 The data registers associated with each port for input and output are 32 bit
23 with the upper 16 bits unused. The output buffer must be written as a 32 bit
24 word, but individual bits may be set or reset separately in atomic operations
25 to avoid race conditions during interrupts. Bits may also be individually
26 locked to prevent accidental configuration changes. Once locked the
27 configuration cannot be changed until after the next reset.
28 
29 Each port bit can be configured as analog or digital input, the latter can be
30 floating or pulled up or down. As outputs they can be configured as either
31 push-pull or open drain, digital I/O or alternate function, and with maximum
32 output speeds of 2MHz, 10MHz, or 50MHz.
33 
34 On reset all ports are configured as digital floating input.
35

82 /** @brief Set GPIO Pin Mode
83 
84 Sets the mode (input/output) and configuration (analog/digitial and
85 open drain/push pull), for a set of GPIO pins on a given GPIO port.
86

89 @param[in] cnf Unsigned int8. Pin configuration @ref gpio_cnf

91  If multiple pins are to be set, use logical OR '|' to separate
92  them.
93 */

95 void gpio_set_mode(uint32_t gpioport, uint8_t mode, uint8_t cnf, uint16_t gpios)
96 {
97  uint16_t i, offset = 0;
98  uint32_t crl = 0, crh = 0, tmp32 = 0;
99 
100  /*
101  * We want to set the config only for the pins mentioned in gpios,
102  * but keeping the others, so read out the actual config first.
103  */
104  crl = GPIO_CRL(gpioport);
105  crh = GPIO_CRH(gpioport);
106 
107  /* Iterate over all bits, use i as the bitnumber. */
108  for (i = 0; i < 16; i++) {
109  /* Only set the config if the bit is set in gpios. */
110  if (!((1 << i) & gpios)) {
111  continue;
112  }
113 
114  /* Calculate bit offset. */
115  offset = (i < 8) ? (i * 4) : ((i - 8) * 4);
116 
117  /* Use tmp32 to either modify crl or crh. */
118  tmp32 = (i < 8) ? crl : crh;
119 
120  /* Modify bits are needed. */
121  tmp32 &= ~(0xf << offset); /* Clear the bits first. */
122  tmp32 |= (mode << offset) | (cnf << (offset + 2));
123 
124  /* Write tmp32 into crl or crh, leave the other unchanged. */
125  crl = (i < 8) ? tmp32 : crl;
126  crh = (i >= 8) ? tmp32 : crh;
127  }
128 
129  GPIO_CRL(gpioport) = crl;
130  GPIO_CRH(gpioport) = crh;
131 }

147 /** @brief Map Alternate Function Port Bits (Main Set)
148 
149 A number of alternate function ports can be remapped to defined alternative
150 port bits to avoid clashes in cases where multiple alternate functions are
151 present. Refer to the datasheets for the particular mapping desired. This
152 provides the main set of remap functionality. See @ref gpio_secondary_remap for
153 a number of lesser used remaps.
154 
155 The AFIO remapping feature is used only with the STM32F10x series.
156 
157 @note The Serial Wire JTAG disable controls allow certain GPIO ports to become
158 available in place of some of the SWJ signals. Full SWJ capability is obtained
159 by setting this to zero. The value of this must be specified for every call to
160 this function as its current value cannot be ascertained from the hardware.
161 
162 @param[in] swjdisable Unsigned int8. Disable parts of the SWJ capability @ref
163 afio_swj_disable.
164 @param[in] maps Unsigned int32. Logical OR of map enable controls from @ref
165 afio_remap, @ref afio_remap_can1, @ref afio_remap_tim3, @ref afio_remap_tim2,
166 @ref afio_remap_tim1, @ref afio_remap_usart3. For connectivity line devices
167 only @ref afio_remap_cld are also available.
168 */
169 void gpio_primary_remap(uint32_t swjdisable, uint32_t maps)
170 {
171  AFIO_MAPR |= (swjdisable & AFIO_MAPR_SWJ_MASK) | (maps & 0x1FFFFF);
172 }
